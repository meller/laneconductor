# Tests: Track 1052 — Show HN Post

## Test Cases

### Phase 1: Post Quality
- [ ] TC-1: Title starts with "Show HN:" and leads with a technical problem (not product name)
- [ ] TC-2: Body mentions filesystem-as-API or context-driven development
- [ ] TC-3: Body is ≤3 paragraphs — no walls of text
- [ ] TC-4: GitHub link present (not landing page)
- [ ] TC-5: No marketing superlatives ("best", "revolutionary", "game-changing")
- [ ] TC-6: No cross-links to other communities

### Phase 2: Post Live
- [ ] TC-7: Submitted to https://news.ycombinator.com/submit
- [ ] TC-8: Post URL saved and accessible
- [ ] TC-9: GIF or demo link present in body (optional but helpful)

### Phase 3: Engagement
- [ ] TC-10: First comment reply within 1 hour
- [ ] TC-11: All technical questions answered honestly
- [ ] TC-12: At least 1 upvote within 2 hours

## Acceptance Criteria
- [ ] Post live on HN with correct GitHub URL
- [ ] Technical framing — no AI hype
- [ ] Local-first angle present
- [ ] Comments engaged promptly
